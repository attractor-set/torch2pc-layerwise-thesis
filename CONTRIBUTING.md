# Участие в проекте

[English version](CONTRIBUTING_EN.md)

## Научная добросовестность

Изменения должны сохранять разделение между выбором по validation и итоговой
оценкой по test. Финальная конфигурация не изменяется после просмотра test без
создания нового явно разведочного эксперимента.

## Рабочий процесс

1. Создать Issue.
2. Создать отдельную ветку.
3. Добавить тесты и обновить документацию.
4. Выполнить `make lint typecheck test validate`.
5. Открыть Pull Request.
6. Связать изменение результатов с экспериментом или ADR.

## Рекомендуемые префиксы commit

- `research:` исследовательский вопрос или гипотеза;
- `experiment:` конфигурация или запуск;
- `analysis:` статистический или визуальный анализ;
- `writing:` текст диссертации или статьи;
- `infra:` Docker, ROCm, CI и окружение;
- `docs:` документация;
- `fix:` исправление без изменения научного вывода.

## Результаты

Не добавляйте в Git скачанные наборы данных, PDF статей, тяжелые checkpoints
или приватные комментарии. Агрегированные результаты добавляются только вместе
с конфигурацией и манифестом соответствующего запуска.
