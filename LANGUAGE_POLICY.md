# Языковая политика проекта

[English version](LANGUAGE_POLICY_EN.md)

## Основной язык

Основной язык репозитория, документации, Issues, Pull Request и диссертации -
русский.

## Английские версии

Английская версия пользовательского документа получает суффикс `_EN`:

```text
README.md              -> README_EN.md
STATUS.md              -> STATUS_EN.md
docs/methodology.md    -> docs/methodology_EN.md
article/README.md      -> article/README_EN.md
```

## Исключения

Следующие элементы сохраняют стандартные технические имена:

- Python-модули, функции, классы и поля конфигурации;
- `LICENSE`, поскольку GitHub распознает стандартное имя;
- `CITATION.cff`, поскольку формат стандартизирован;
- `Dockerfile`, `Makefile`, `compose.yaml`, `pyproject.toml`;
- ключи GitHub Actions и YAML;
- библиографические ключи BibTeX;
- английская рукопись статьи `manuscript_EN.tex`.

## Правила обновления

1. Сначала обновляется русская версия.
2. Английская версия обновляется в том же Pull Request.
3. Если английская версия временно отсутствует, это отмечается в Issue.
4. Автоматическая проверка `scripts/check_language_structure.py` контролирует
   обязательные пары.
5. Научные данные, численные значения и ссылки должны совпадать между версиями.

## Язык исходного кода

Идентификаторы и API остаются английскими для совместимости с Python,
Torch2PC и научной экосистемой. Пользовательские инструкции и документация
остаются русскими.
