# Предварительная спецификация анализа

[English version](PREREGISTRATION_EN.md)

Статус: **черновик до выполнения control gate и pilot**.

## Основная статистическая единица

Независимо обученная модель (`model_seed`).

## Основной набор данных

FashionMNIST. MNIST используется как вторичный и контрольный набор.

## Основная метрика

Macro F1 на final test после выбора конфигурации по validation.

## Основные сравнения

- FixedPred против BP;
- Strict против BP.

Exact используется прежде всего как технический контроль связи с BP. Его
итоговые метрики сохраняются, но не входят в основное семейство сравнений.

## Число повторов

Минимум 10 парных `model_seed`: 0-9. После pilot число повторов может быть
увеличено по заранее описанному precision/power правилу, но не уменьшено после
просмотра test.

## Test policy

- smoke: test недоступен;
- pilot: test недоступен;
- final: test оценивается после pilot-freeze;
- изменение конфигурации после просмотра test создает новый exploratory анализ.

## Эквивалентность

Рабочая граница для парного различия macro F1: 0.01. Эквивалентность отмечается
только если 90% CI парного различия полностью находится внутри `[-0.01, 0.01]`.

## Различие

Для парных различий используется exact sign-flip test при допустимом числе пар
или заранее описанная аппроксимация. P-values основного семейства корректируются
методом Holm. Одновременно публикуются парное среднее различие, 95% CI и Cohen dz.

## Пропуски и ошибки

- все planned runs остаются в реестре;
- success rate публикуется отдельно;
- complete-pair анализ сопровождается sensitivity-анализом при наличии ошибок;
- технические исключения описываются до повторного запуска.

## Заморозка

Перед final создается `pilot-freeze_manifest.json`, содержащий Git commit и
SHA-256 основных конфигураций.

## Правило выбора pilot

Параметры FixedPred и Strict ранжируются только по FashionMNIST validation.
Порядок критериев: success rate, средний validation macro F1, среднее время
обучения, число итераций вывода, eta. MNIST не влияет на выбор и используется
как вторичная проверка.

При наличии менее десяти полных final-пар анализ маркируется как неполный и не
используется для вывода об эквивалентности.
