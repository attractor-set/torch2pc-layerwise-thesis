# Предварительная спецификация анализа

[English version](PREREGISTRATION_EN.md)

Статус: **предварительно специфицировано до запуска pilot и до доступа к final test**.
Изменения после этой точки оформляются отдельным ADR с указанием времени,
причины и затронутых выводов. Изменения после доступа к final test считаются
разведочными и не заменяют исходный подтверждающий анализ.

## Статистическая единица

Независимо обученная модель, определяемая уникальным `model_seed`. Пакеты,
объекты, параметры и слои не считаются независимыми повторностями модели.

## Основной дизайн

- основной набор данных: FashionMNIST;
- вторичный набор данных: MNIST;
- основная архитектура: `lenet_classic`;
- основная метрика: macro F1;
- основные контрасты: FixedPred против BP и Strict против BP;
- Exact используется как технический и вторичный сравнительный режим.

## Число повторов

Исходный минимум: 10 полных пар по `model_seed` 0-9.

До доступа к final test допускается только увеличение числа пар по следующему
заранее заданному advisory-правилу. Для каждого основного контраста по трем
pilot seed оценивается стандартное отклонение парной разности validation macro
F1. Ориентировочное число пар рассчитывается как
`ceil((1.96 * SD / 0.01)^2)` и ограничивается диапазоном 10-30. Для final
используется максимум оценок по двум основным контрастам. Это правило является
планированием точности по pilot validation, а не гарантией мощности. Число пар
не уменьшается после просмотра pilot или final test.

## Доступ к test

- correctness и smoke: test не создается;
- pilot: test не создается;
- final: test оценивается только после `pilot-freeze`;
- изменение конфигурации после просмотра test создает новый exploratory ID.

## Эквивалентность

Практически значимая граница парной разности macro F1 фиксируется до pilot и
равна 0.01 в абсолютных единицах. Эквивалентность отмечается только при наличии
не менее десяти полных пар и если 90% CI парной разности полностью находится
внутри `[-0.01, 0.01]`.

Отсутствие статистически обнаруженного различия не интерпретируется как
эквивалентность.

## Анализ различий

Для каждой основной парной разности публикуются:

- значения по каждому seed;
- средняя парная разность;
- 95% CI;
- Cohen dz;
- exact sign-flip test при числе ненулевых пар не более 20;
- заранее реализованная двусторонняя аппроксимация Wilcoxon при большем числе;
- Holm-коррекция для двух основных контрастов.

При наличии менее десяти полных пар оценки сохраняются как описательные;
Holm-скорректированный подтверждающий вывод и вывод об эквивалентности не
формируются.

## Неудачные и повторные запуски

- все запланированные ячейки сохраняются в append-only реестре;
- success rate рассчитывается по первой терминальной попытке каждой
  конфигурации/seed;
- последующий успешный повтор не превращает исходную неудачу в успех для
  pilot-ранжирования;
- технический повтор сохраняет тот же экспериментальный смысл и не считается
  новой независимой репликацией;
- причина повторного запуска документируется.

## Выбор pilot

Параметры FixedPred и Strict выбираются только по FashionMNIST validation для
одной заранее указанной архитектуры. Критерии применяются в следующем порядке:

1. более высокий success rate;
2. более высокий средний validation macro F1;
3. меньшее среднее измеренное время обучающих эпох;
4. меньшее число итераций вывода;
5. меньшее eta.

MNIST не участвует в ранжировании и описывается как вторичная проверка.
Измеренное время используется только как tie-breaker и не заменяет отдельный
equal-wall-clock эксперимент.

## Заморозка

Перед final создается `pilot-freeze_manifest.json`, который фиксирует:

- Git commit проекта;
- SHA-256 environment lock;
- выбранные параметры методов;
- SHA-256 базовой, методических и final-конфигураций;
- SHA-256 этой спецификации, гипотез и плана анализа.


## Дополнение для Stage 3

Исходная спецификация выше относится к завершённым Stage 1/2 и не
переписывается. Stage 3 создаёт отдельный protocol/freeze. Profiling и pilot не
создают test loader. Exact implementation candidates требуют equivalence gates;
approximation candidates получают отдельный non-inferiority rule и не получают
claim эквивалентности Stage 2. Candidate selection, stopping tolerance, refresh interval, predict-correct correction budget, EMA/secant parameters, fallback rule и margin замораживаются до Stage 3 test access. A0 использует отдельный endpoint-equivalence gate; C4/C5 требуют хотя бы одну точную коррекцию и публикуемый fallback rate.
