# План анализа

[English version](analysis-plan_EN.md)

## Основной анализ

В качестве основного набора данных выбран FashionMNIST, основной метрики -
macro F1, а статистической единицы - независимо обученная модель. Основные
парные сравнения: FixedPred с BP и Strict с BP. Для каждого сравнения
планируется не менее десяти полных пар по одинаковым `model_seed`.

Для каждого основного сравнения будут опубликованы:

- исходные значения по каждому seed;
- среднее парное различие;
- 95% доверительный интервал;
- Cohen dz;
- парный exact sign-flip test либо заранее описанная аппроксимация;
- скорректированное значение после процедуры Holm;
- отдельная оценка эквивалентности.

## Эквивалентность

Эквивалентность не выводится из отсутствия статистически обнаруженного
различия. Она оценивается отдельно: 90% доверительный интервал парного различия
macro F1 должен полностью находиться внутри заранее зафиксированной границы.
Рабочая граница 0.01 фиксируется до запуска pilot и не изменяется после
получения pilot или final результатов.

## Вторичный анализ

К вторичным результатам относятся MNIST, accuracy, loss, траектория обучения,
режим Exact, success rate и вычислительные метрики. Вторичный анализ маркируется
отдельно и не заменяет основной.

## Послойный анализ

Градиентные показатели сначала агрегируются внутри одной обученной модели, а
затем между model seeds. Отдельные параметры, слои и пакеты не считаются
независимыми повторностями модели.

## Анализ представлений

Для CKA и RSA неопределенность должна включать вариацию между model seeds.
Bootstrap только по изображениям считается недостаточным. Планируется
иерархический bootstrap либо сравнение seed-level оценок.

## Устойчивость

Один и тот же checkpoint оценивается на детерминированных искажениях с
фиксированными corruption seeds. Будут рассчитаны чистая метрика, относительная
деградация, наклон по severity и значение на максимальной severity.

## Вычислительная стоимость

Сравнение по одинаковому числу обновлений и по одинаковому реальному времени
проводится раздельно. Измерения предусматривают прогрев, синхронизацию GPU,
детерминированно контрбалансированный порядок методов, несколько повторов и
запись состояния хоста до и после серии. Постпилотная фиксация порядка и
телеметрии описана в `docs/decisions/ADR-005-post-pilot-final-execution.md`.

## Полнота подтверждающего анализа

Основной контраст получает статус `confirmatory` только при наличии не менее
десяти полных пар по заранее заданным seed. При меньшем числе пар публикуются
описательные оценки, но p-value Holm и вывод об эквивалентности не формируются.

Pilot-конфигурация выбирается только по FashionMNIST validation. MNIST
используется как вторичная проверка переносимости выбранных параметров и не
участвует в ранжировании кандидатов.
