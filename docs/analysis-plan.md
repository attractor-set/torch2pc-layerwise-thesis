# План анализа

[English version](analysis-plan_EN.md)

## Основной анализ

Основным набором данных выбран FashionMNIST, основной метрикой — макро-F1, а
независимой статистической единицей — отдельно обученная модель. Основные
парные сравнения: FixedPred с BP и Strict с BP. Для каждого сравнения
планируется не менее десяти полных пар с одинаковыми значениями `model_seed`.

Для каждого основного сравнения публикуются:

- исходные значения для каждой отдельно обученной модели;
- среднее парное различие;
- 95%-й доверительный интервал;
- `Cohen dz`;
- точная парная перестановочная проверка со сменой знака либо заранее
  описанная аппроксимация;
- скорректированное значение после процедуры Холма;
- отдельная оценка эквивалентности.

## Эквивалентность

Эквивалентность не выводится из отсутствия статистически обнаруженного
различия. Она оценивается отдельно: 90%-й доверительный интервал парного
различия макро-F1 должен полностью находиться внутри заранее зафиксированной
границы. Рабочая граница 0,01 фиксируется до пилотного исследования и не
изменяется после получения пилотных или итоговых результатов.

## Вторичный анализ

К вторичным результатам относятся MNIST, точность, функция потерь, траектория
обучения, режим Exact, доля успешных запусков и вычислительные метрики.
Вторичный анализ маркируется отдельно и не заменяет основной.

## Послойный анализ

Градиентные показатели сначала агрегируются внутри одной обученной модели, а
затем между моделями с разными значениями `model_seed`. Отдельные параметры,
слои и пакеты не считаются независимыми повторениями модели.

## Анализ представлений

Для CKA и RSA неопределённость должна учитывать вариативность между отдельно
обученными моделями. Повторная выборка только по изображениям считается
недостаточной. Планируется иерархическая повторная выборка либо сравнение
оценок, рассчитанных на уровне модели.

## Устойчивость

Одна и та же [контрольная точка](glossary.md#term-checkpoint) оценивается на детерминированных искажениях с
фиксированными случайными начальными значениями генератора искажений.
Рассчитываются метрика на чистых данных, относительная деградация, наклон по
тяжести искажения и значение при максимальной тяжести.

## Вычислительная стоимость

Сравнение при одинаковом числе обновлений и при одинаковом астрономическом
времени проводится раздельно. Измерения предусматривают [разогрев](glossary.md#term-warm-up),
синхронизацию GPU, детерминированно уравновешенный порядок методов, несколько
повторений и запись состояния узла до и после серии. Постпилотная фиксация
порядка и телеметрии описана в
`docs/decisions/ADR-005-post-pilot-final-execution.md`.

## Полнота подтверждающего анализа

Основной контраст получает статус `confirmatory` только при наличии не менее
десяти полных пар с заранее заданными значениями `model_seed`. При меньшем
числе пар публикуются описательные оценки, но p-значение после поправки Холма и
вывод об эквивалентности не формируются.

Пилотная [конфигурация](glossary.md#term-configuration) выбирается только по валидационной выборке FashionMNIST.
MNIST используется как вторичная проверка переносимости выбранных параметров и
не участвует в ранжировании кандидатов.

## Дополнение этапа 3

Этап 3 не изменяет [подтверждающий анализ](glossary.md#term-confirmatory-analysis) этапов 1 и 2. Он использует отдельные
единицы анализа:

- качество обучения: отдельно обученная модель;
- [профилирование](glossary.md#term-profiling): сопоставленная [экспериментальная ячейка](glossary.md#term-experiment-cell) или повторение
  внутри аппаратного блока;
- локальность: событие сначала агрегируется внутри запуска, затем по отдельно
  обученным моделям;
- согласованность градиентов: слой агрегируется внутри модели до анализа между
  моделями.

B1 и B2 оцениваются с помощью проверки численной эквивалентности и парных
наблюдений времени выполнения и памяти. C1 и C2 оцениваются по критерию не
меньшей эффективности на валидационной выборке, согласованности градиентов и
сокращению вычислений. Итоговая граница не меньшей эффективности фиксируется до
доступа к тестовой выборке этапа 3. Подробности:
[протокол этапа 3](stage-3-protocol.md).
