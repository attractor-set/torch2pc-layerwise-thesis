# ADR-005: постпилотная фиксация итогового выполнения

- Статус: принято
- Дата: 2026-07-13
- Момент принятия: после пилотного исследования только на валидационной
  выборке, до `pilot-freeze` и до любого доступа к итоговой тестовой оценке

## Контекст

[Пилотное исследование](../glossary.md#term-pilot-study) завершило 96 из 96 запланированных ячеек без неудач и
выбрало параметры `FixedPred(eta=0.1, n=10)` и
`Strict(eta=0.05, n=20)`. Расчёт требуемой точности дал рекомендательное
значение 10 полных пар. Поэтому число случайных начальных значений модели для
итогового выполнения остаётся равным 10 (`0-9`) в соответствии с
предварительной спецификацией и не увеличивается по направлению пилотного
эффекта.

До итогового выполнения были обнаружены инфраструктурные ограничения, не
меняющие научные гипотезы: порядок методов был блочным, полный реестр
одновременно использовался как набор пилотных доказательных материалов и как
изменяемый журнал итогового выполнения, а файл фиксации среды не содержал
сводного хэша дерева конфигураций.

## Решение

1. Создавать до `pilot-freeze` неизменяемый `final_execution_plan.json`.
2. Уравновешивать порядок методов детерминированным ранжированием по SHA-256
   отдельно для каждой комбинации набора данных, модели и `model_seed`.
3. Сохранять отдельный `pilot_registry_snapshot.csv`, а общий
   `experiments/registry.csv` оставлять журналом итогового выполнения, в
   который записи только добавляются.
4. Разрешать безопасное возобновление итогового выполнения: завершённые ячейки
   текущей когорты пропускаются, неудачные остаются в реестре и могут быть
   технически повторены.
5. Синхронизировать GPU перед каждой измеряемой эпохой и сохранять среднее и
   медианное время эпохи, устройство, версии PyTorch/HIP и пиковую память GPU.
6. Добавить сводный SHA-256 дерева конфигураций в файл фиксации среды.

## Неизменяемые части дизайна

- основной [набор данных](../glossary.md#term-dataset): FashionMNIST;
- дополнительный [набор данных](../glossary.md#term-dataset): MNIST;
- основная метрика: макро-F1;
- основные контрасты: FixedPred с BP и Strict с BP;
- граница эквивалентности: `+-0.01`;
- случайные начальные значения итогового выполнения: `0-9`;
- выбранные гиперпараметры;
- одноразовая тестовая оценка каждого завершённого итогового запуска.

## Последствия

Изменения повышают воспроизводимость порядка выполнения, корректность
возобновления и качество вычислительной телеметрии. Они не являются повторным
выбором гиперпараметров и не меняют подтверждающие гипотезы. Сравнение времени
обучения остаётся вторичным описательным результатом; отдельный анализ при
одинаковом астрономическом времени выполняется только как заранее маркированный
диагностический этап.
