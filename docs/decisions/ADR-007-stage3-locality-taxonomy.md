# ADR-007: многомерная таксономия локальности

[English version](ADR-007-stage3-locality-taxonomy_EN.md)

- Статус: принято для проектирования
- Дата: 2026-07-13

## Контекст

Torch2PC использует послойные ошибки предсказания и соседние зависимости в
формулах, но выполняет их с помощью централизованного планировщика Python и
автоматического дифференцирования PyTorch. Один бинарный признак
«локально/нелокально» не описывает этот разрыв.

## Решение

Этап 3 публикует отдельные измерения:

- множество алгоритмических зависимостей;
- радиус зависимостей;
- модули и протяжённость вычислительного графа;
- время жизни независимого графа;
- тип оператора обратной связи;
- оркестрацию и точки синхронизации;
- число вызовов VJP и объём сохранённых тензоров.

Структурная проверка `dependency_radius <= 1` применяется только к событиям,
объявленным математически локальными по слоям. Составной VJP может иметь
большую протяжённость графа и при этом сохранять локальное правило обновления.

## Последствия

Локальность не сводится к одной оценке. Сравнение B1 и B2 явно представляет
соотношение между локальностью выполнения и пропускной способностью.
Приближённая обратная связь получает отдельную метку изменения алгоритма.
