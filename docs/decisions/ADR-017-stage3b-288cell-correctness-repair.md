# ADR-017: исправление корректности перед кампанией этапа 3B из 288 ячеек

[English version](ADR-017-stage3b-288cell-correctness-repair_EN.md)

- Статус: принято
- Дата: 2026-07-19

## Контекст

Предстартовый аудит сопоставленной кампании B0/B1/B2 выявил четыре независимых
риска: несовместимость `retry/resume` с последующей фиксацией, допуск по
`smoke`-эквивалентности вместо подтверждающих доказательных материалов,
несбалансированный порядок трёх кандидатов; каждый [кандидат](../glossary.md#term-candidate) для
`FixedPred` и отсутствие отдельной проверки эквивалентности кандидатов на
фактической матрице масштабирования.

Две предыдущие попытки показали, что только авторизация среды выполнения и
успешная сухая проверка недостаточны для доказательной кампании.

## Решение

1. История попыток остаётся только дополняемой.
2. Повтор разрешён только для явно классифицированных инфраструктурных или
   операторских прерываний. Неизвестные, научные ошибки и ошибки корректности
   повторять нельзя.
3. Фиксация допускает разрешённую для повтора ошибку или прерывание перед одной
   финальной успешной попыткой, но отклоняет запрещённую для повтора ошибку,
   несколько успешных попыток, незавершённую попытку или успех, который не
   является последней попыткой.
4. Производственное [выполнение](../glossary.md#term-execution) требует подтверждающий `EQ-B1` с `120/120`
   парами и подтверждающий `EQ-B2` с `120/120` тройками и `240/240`
   сравнениями. Инженерные проверки на CPU не открывают производственный допуск.
5. Для каждого метода используются все шесть перестановок трёх кандидатов по
   восемь раз: каждый кандидат занимает каждую позицию 16 раз, а каждое
   попарное предшествование встречается 24 раза.
6. Перед измерениями каждого блока выполняется отдельная неизмеряемая
   межкандидатная проверка корректности. Она сравнивает снимок без наблюдателя
   для `stage2_baseline` со снимками `isolated_layer_vjp` и `composite_vjp` при
   одинаковых начальном состоянии модели и синтетическом мини-пакете.
7. Ошибка корректности не допускает повтора, немедленно блокирует кампанию и
   требует нового коммита исходного кода и нового корня результатов.
8. Фиксация требует запись межкандидатной корректности для каждого блока и
   сохраняет историю попыток и записи корректности в доказательном пакете.

## Последствия

- ранее выпущенные конверты авторизации и корни результатов нельзя использовать
  после изменения дайджеста манифеста, исходного кода или образа;
- текущие решения `smoke` остаются неизменяемыми историческими артефактами, но
  не открывают производственный [запуск](../glossary.md#term-run);
- подтверждающие B1/B2 должны быть выполнены и запечатаны до новой фиксации
  среды;
- измерение стоимости наблюдателя и проверка корректности остаются разными
  контурами: проверка не входит в основное измерение времени;
- восстановление сохраняется только для явно разрешённых к повтору событий.

## Критерий GO

Кампания из 288 ячеек разрешена только после одновременного прохождения:
подтверждающей эквивалентности, точной балансировки порядка, неизмеряемой
межкандидатной проверки, интеграционной проверки повтора и фиксации, нового
неизменяемого образа, новой фиксации среды, предварительной проверки,
авторизации и сухой проверки на полностью пустом корне результатов.
