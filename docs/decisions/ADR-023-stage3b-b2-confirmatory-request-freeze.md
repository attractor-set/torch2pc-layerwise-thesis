# ADR-023: фиксация запроса подтверждающего `EQ-B2`

[English version](ADR-023-stage3b-b2-confirmatory-request-freeze_EN.md)

- Статус: принято — запрос заморожен, [выполнение](../glossary.md#term-execution) закрыто
- Дата: 2026-07-20

## Контекст

[ADR-021](ADR-021-stage3b-b2-confirmatory-preregistration.md) предварительно
зарегистрировал подтверждающий `EQ-B2` как 120 сопоставленных троек и 240
прямых сравнений. [ADR-022](ADR-022-stage3b-b2-confirmatory-opening.md)
реализовал закрытую при ошибке инфраструктуру планирования, авторизации,
восстановления и фиксации результатов, но намеренно не создавал неизменяемый
запрос.

До сборки образа и проверки [среды выполнения](../glossary.md#term-runtime)
необходима отдельная предварительная фиксация запроса, связывающая кампанию с
уже зарегистрированными входами B1 и контрактами B2, не разрешая измерения.

## Решение

Зафиксировать неизменяемый пакет:

```text
experiments/frozen/stage3b-b2-confirmatory/request.json
experiments/frozen/stage3b-b2-confirmatory/SHA256SUMS
```

Канонические идентификаторы:

```text
request_id=stage3b-b2-confirmatory-120-v1
request_sha256=02c88c0778398cecedb069948e9f952c8d7be46d3d064197ae0e7f5b86314ed3
request_digest=3a824f4a7a8517a4b97341a4598ac8433e2253b45d2bb3f667722c270b06fb78
matched_triple_count=120
pairwise_comparison_count=240
run_seed_base=732000
```

Запрос повторно использует без изменения байтов:

- три контрольные точки подтверждающего B1;
- десять пакетов валидационной выборки B1;
- запечатанное решение `EQ-B1-CONFIRMATORY`;
- производный допуск B1;
- предварительную регистрацию и контракты реализации B2.

Новый выбор данных не выполняется. [Доступ к тестовой выборке](../glossary.md#term-test-dataset-access) остаётся закрытым.

## Граница выполнения

После слияния истинно только:

```text
b2_confirmatory_request_frozen=true
runtime_authorization=not_issued
execution_started=false
results_present=false
measurements_allowed=false
```

Замороженный запрос не является [доказательным материалом](../glossary.md#term-evidence),
не разрешает публикацию результатов и не создаёт токен авторизации.

Следующий отдельный этап должен:

1. собрать неизменяемый образ ROCm из чистой публикационной фиксации;
2. проверить метку происхождения образа, связывающую его с исходной фиксацией, и оба вычислительных контура;
3. выпустить отдельное закрытое при ошибке разрешение выполнения;
4. выполнить только неизмеряемый `dry-run` до отдельного решения об инженерной проверке или подтверждающем выполнении.

## Неизменяемые границы

Этот ADR не изменяет:

- старую инженерную `smoke`-проверку B2;
- подтверждающие артефакты B1;
- прежние запрос и манифест [сопоставленного профилирования](../glossary.md#term-matched-profiling);
- `experiments/planned/STAGE3B-B2-CONFIRMATORY-CONTRACT.json`;
- тестовую выборку;
- любой файл под `results/**`.

Любое будущее изменение запроса требует нового версионированного идентификатора и отдельной записи о замене; существующий `request.json` не переписывается.
