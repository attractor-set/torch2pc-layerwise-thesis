# ADR-025: повторная фиксация запроса сопоставленного профилирования `Stage 3B`

[English version](ADR-025-stage3b-matched-profiling-request-refreeze_EN.md)

- **Статус:** принято
- **Дата:** 20 июля 2026 года

## Контекст

Исторические `v1`-запрос и манифест были сформированы на основе инженерных
проверок B1 и B2. После публикации запечатанных подтверждающих решений
`EQ-B1` и `EQ-B2` прежняя фиксация не получает допуск задним числом и
сохраняется побайтно неизменной.

## Решение

Создать новый неизменяемый пакет:

```text
experiments/frozen/stage3b-matched-profiling-v2/
  SHA256SUMS
  manifest.json
  request.json
```

`request.json` перспективно ссылается на запечатанные подтверждающие допуски
`EQ-B1` и `EQ-B2`, фиксирует историческую версию `v1` по SHA-256 и запрещает
ретроспективный допуск. `manifest.json` сохраняет зарегистрированную матрицу
из 288 ячеек, точную балансировку порядка и закрытую тестовую выборку.

## Граница

```text
scientific_admission=open
matched_profiling_request_refrozen=true
matched_profiling_request_refresh_required=false
matched_profiling_execution_open=false
runtime_authorization=not_issued
measurements_allowed=false
test_dataset_access=false
```

Повторная фиксация не создаёт разрешение [среды выполнения](../glossary.md#term-runtime),
измерения или [доказательные материалы](../glossary.md#term-evidence). Следующий
допустимый переход — отдельный неизменяемый образ, предварительные проверки
ROCm, разрешение среды и неизмеряющий пробный план. Только после этих проверок
может быть разрешена кампания [сопоставленного профилирования](../glossary.md#term-matched-profiling)
из 288 ячеек.

## Последствия

- подтверждающая цепочка допуска B1/B2 становится входом предварительной
  производственной проверки;
- исторические артефакты `v1` остаются проверяемыми и неизменными;
- исполнитель и сценарии среды по умолчанию используют только пакет `v2`;
- `EX-IF0`, оцениватель, `QWake-PC`, офлайн-политика и тестовая выборка
  остаются закрытыми.
