# ADR-028: реализация описательного анализа сопоставленного профилирования

[English version](ADR-028-stage3b-matched-descriptive-analysis-implementation_EN.md)

- **Статус:** принято
- **Дата:** 2026-07-21

## Контекст

ADR-027 зафиксировал `post-collection/pre-analysis` контракт до вычисления
сравнительных результатов. Ранняя версия `stage3b_matched_analysis.py`
создавала только четыре файла, агрегировала конфигурации напрямую и не
реализовывала зарегистрированные правила масштабирования,
[Парето-допустимости](../glossary.md#term-pareto-admissibility), локальности и
решений `retain / conditional / reject_or_revise`.

После слияния ADR-027 разрешена реализация и синтетическая проверка, но
[выполнение](../glossary.md#term-execution) на запечатанных
[доказательных материалах](../glossary.md#term-evidence) остаётся закрытым до
отдельного машиночитаемого допуска.

## Решение

Заменить ранний анализатор зарегистрированным вычислительным ядром, которое:

- проверяет полную матрицу из 288 ячеек, 96 сопоставленных блоков и 1440 строк
  повторов;
- формирует ровно 192 парные строки, 64 конфигурационные строки, четыре
  сводки `candidate × method`, 96 строк Парето-множества, 288 строк локальности
  и 84 строки масштабирования;
- реализует семимерное Парето-правило с обязательной альтернативой
  `stage2_baseline` и точностью `1e-12`;
- применяет пороги 15%, 20%, 3% и 15% только в зарегистрированном порядке;
- потоково читает обычный или `Zstandard`-сжатый поток событий локальности;
- вычисляет шесть зарегистрированных диагностик локальности без загрузки всего
  потока в память;
- создаёт ровно 18 детерминированных файлов, включая шесть `PDF`, два отчёта и
  `SHA256SUMS`;
- проверяет контрольные суммы входов до и после чтения;
- использует атомарный новый каталог выхода и закрывается при ошибке, если
  каталог уже существует;
- сохраняет выход незапечатанным и запрещает публикацию, `EX-IF0`, активацию
  политики и утверждения о превосходстве.

Синтетический путь требует отдельный маркер полной фикстуры. Публичная функция
для запечатанных доказательных материалов намеренно выбрасывает ошибку до
слияния отдельного допуска выполнения. Скрипт командной строки принимает только
синтетический каталог.

## Следствия

Положительные:

- весь контракт ADR-027 проверяется на полной синтетической матрице;
- порядок агрегации «измеряемые шаги → повторы → ячейки → блоки →
  `model_seed` → конфигурации» отражён явно;
- выходы воспроизводимы побайтно при фиксированном времени генерации;
- входы остаются только для чтения;
- будущий допуск выполнения может повторно использовать уже проверенное ядро,
  не меняя оценки или схемы.

Ограничения:

- синтетический выход не является доказательством и не может быть сохранён как
  научный результат;
- обработка полного сжатого потока локальности ещё не запускалась на
  запечатанном источнике;
- реализация не открывает анализ и не создаёт решение по B1/B2;
- перед запуском требуется отдельная фиксация запроса, авторизации и нового
  каталога выхода.
