# Методология

[English version](methodology_EN.md)

## Дизайн

Исследование имеет вычислительно-экспериментальный характер. Сравнение является
парным по `model_seed`, а основной статистической единицей считается независимо
обученная модель.

## Стадии

1. статическая проверка;
2. подготовка ресурсов;
3. структурная проверка выбранных формул;
4. C0/C1 на нескольких seed и пакетах;
5. pilot без test;
6. заморозка конфигурации;
7. final с test;
8. послойная и вычислительная диагностика;
9. статистическая оценка и интерпретация.

## Наборы данных

- FashionMNIST - основной набор;
- MNIST - вторичный контрольный набор;
- KMNIST - возможное расширение после основного анализа.

## Метрики

Primary: macro F1. Secondary: accuracy, loss, success rate, wall-clock time,
peak memory, gradient cosine, relative L2, CKA и RSA.

## Выбор конфигурации

Pilot использует только validation. Test dataset не загружается. Выбор
FixedPred/Strict производится по заранее указанной метрике с техническими
tie-breakers. После pilot создается манифест заморозки.

## Final

Final использует заранее заданные парные seed. Test оценивается после загрузки
best-validation checkpoint. Любое изменение конфигурации после test относится к
новому exploratory анализу.

## Статистика

Публикуются исходные парные значения, среднее различие, 95% CI, Cohen dz,
парный sign-flip/Wilcoxon test и Holm adjustment. Эквивалентность оценивается
отдельно по 90% CI и заданной границе.

## Ограничения

Результат может зависеть от commit Torch2PC, dtype, optimizer, architecture,
dataset, seed policy и аппаратной среды. Эти зависимости рассматриваются как
часть вывода, а не как скрытые неудобства.
