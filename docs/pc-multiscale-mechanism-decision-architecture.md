# Многомасштабная механизмно-решающая архитектура `predictive coding`

[English version](pc-multiscale-mechanism-decision-architecture_EN.md)

## 1. Статус и граница

[Многомасштабная механизмно-решающая архитектура](glossary.md#term-multiscale-mechanism-decision-architecture)
соединяет `PC-CATM`, `PC-TREF` и `QWake-PC` в единую композиционную схему.
Документ является теоретическим и проектным расширением. Он не разрешает
[выполнение](glossary.md#term-execution), не изменяет замороженные контракты
B1/B2, не открывает тестовую выборку и не делает многомасштабный контроллер
обязательным условием `A-Core`.

Текущая работа сохраняет непрерывные состояния и ошибки базовой PC-сети.
Спайкоподобность относится только к организации пороговых, разреженных во
времени и избирательных по масштабу событий коррекции.

## 2. Множество масштабов

Пусть

```math
\mathcal S=(S,\preceq)
```

— конечное частично упорядоченное множество вычислительных масштабов. Запись
$s\preceq t$ означает, что $s$ тоньше $t$. Возможная иерархия:

```text
координата состояния
→ группа координат или канал
→ блок
→ слой
→ полная сеть
```

Не каждая реализация обязана поддерживать все масштабы. Масштаб входит в
исполняемую архитектуру только после фиксации его состояния, нормы, правил
агрегации, пространства действий, стоимости и точной проверки.

## 3. Масштабно-зависимая модель `PC-CATM`

Для каждого $s\in S$ задаётся

```math
\mathcal M_s=
(\mathcal X_s,\mathcal C_s,\mathcal T_s,\mathcal G_s,
\|\cdot\|_s,\chi_s),
```

где:

- $\mathcal X_s$ — пространство состояний;
- $\mathcal C_s$ — семейство канонических каналов коррекции;
- $\mathcal T_s$ — операторы переноса ошибки;
- $\mathcal G_s$ — оператор агрегации каналов;
- $\|\cdot\|_s$ — зарегистрированная норма;
- $\chi_s$ — классификатор механизмного режима.

Для состояния $x_s$ результирующая коррекция определяется как

```math
u_s(x_s)=
\mathcal G_s
\left(
T_s^{(1)}c_s^{(1)},\ldots,T_s^{(m_s)}c_s^{(m_s)}
\right).
```

Классификатор может возвращать `NCZ`, `ECZ`, `TNZ`, `active`, `unresolved` или
`invalid`. Эти категории описывают механизм и сами по себе не разрешают
вычислительное действие.

## 4. Диагностическое представление и достаточность

Для каждого масштаба задаётся представление

```math
\phi_s:\mathcal X_s\times\mathcal H_s\rightarrow\mathcal Z_s,
```

которое может включать геометрию каналов, перенос, компенсацию, остаток,
временную устойчивость, неопределённость, покрытие и стоимость признаков.

Пусть $\mathcal A_s$ — зарегистрированное пространство действий, а
$R_s(a,x_s)$ — потери решения для действия. Множество допустимых действий:

```math
\mathcal A_s^*(x_s)=
\{a\in\mathcal A_s:
R_s(a,x_s)\leq\varepsilon_s
\land Safe_s(a,x_s)\}.
```

Диагностическое представление достаточно для решения на масштабе $s$, если

```math
\phi_s(x_s)=\phi_s(y_s)
\Longrightarrow
\mathcal A_s^*(x_s)=\mathcal A_s^*(y_s).
```

Это масштабно-зависимая специализация `PC-TREF`. Достаточность на одном
масштабе не переносится автоматически на другой.

## 5. Межмасштабная композиция

Для $s\preceq t$ задаются отображения укрупнения состояния и каналов:

```math
P_{s\rightarrow t}:\mathcal X_s\rightarrow\mathcal X_t,
```

```math
A_{s\rightarrow t}:\mathcal C_s\rightarrow\mathcal C_t.
```

[Дефект композиции коррекции](glossary.md#term-correction-composition-defect)
определяется как

```math
\Delta^u_{s\rightarrow t}(x_s)=
\left\|
P^u_{s\rightarrow t}(u_s(x_s))
-
u_t(P_{s\rightarrow t}(x_s))
\right\|_t.
```

Межмасштабное утверждение требует заранее заданного допуска
$\tau^u_{s\rightarrow t}$ и проверки

```math
\Delta^u_{s\rightarrow t}(x_s)
\leq\tau^u_{s\rightarrow t}.
```

Без такой проверки недопустимы выводы

```math
NCZ_s \Rightarrow NCZ_t,
```

```math
ECZ_s \Rightarrow ECZ_t,
```

или перенос локального разрешения на остановку к полной сети.

## 6. Политика `QWake-PC` над масштабом, действием и бюджетом

Объект выбора контроллера — `scale, action, budget`.

[QWake-PC](glossary.md#term-qwake-pc) получает доступные представления

```math
\Phi_t=\{\phi_s(x_{s,t},h_{s,t}):s\in S_t\}
```

и предлагает решение

```math
\pi_Q:(\Phi_t,H_t)\rightarrow(s_t,a_t,k_t),
```

где $s_t$ — масштаб действия, $a_t\in\mathcal A_{s_t}$ — действие, а $k_t$ —
вычислительный бюджет.

Примеры проектных действий:

```text
stop
local_sweep(block_id)
layer_sweep(layer_id)
full_exact
fallback_exact
```

Наличие действия в проектном пространстве не означает его допуска. Пара
«действие–бюджет» допустима только при положительных проверках покрытия,
достаточности, точной контрфактической проверки, потерь решения, безопасности и полной
стоимости.

## 7. Консервативная эскалация

Если на масштабе $s$ нет допустимого действия, выполняется
[адаптивная эскалация](glossary.md#term-adaptive-escalation) к более крупному
масштабу:

```text
дешёвая локальная диагностика
→ локальное действие, если оно доказательно допустимо
→ более крупный масштаб при недостаточности
→ fallback_exact при небезопасной неопределённости
```

Формально эскалация требуется, если

```math
\mathcal Q_s(\Phi_t,H_t)=\varnothing.
```

Недостаточность локальных данных должна приводить к более полному вычислению,
а не к более агрессивному сокращению.

## 8. Спайкоподобная граница текущей работы

[Спайкоподобная управляющая динамика](glossary.md#term-spike-like-control-dynamics)
может включать накопление диагностического сигнала, порог пробуждения,
гистерезис, устойчивость, дискретный бюджет и временную паузу после события.
Она делает вычисления событийными и разреженными, но не изменяет тип динамики
базовых состояний и ошибок.

Каноническая граница текущей работы:

> Многомасштабная архитектура [QWake-PC](glossary.md#term-qwake-pc) приближает `predictive-coding inference`
> к спайкоподобному событийно-управляемому вычислению за счёт пороговых,
> разреженных во времени и избирательных по масштабу событий коррекции,
> сохраняя при этом неспайковую динамику состояний и ошибок базовой сети.

Текущая система поэтому не называется спайковой нейронной сетью.

## 9. Перспективная граница `QWake-SPC`

[`QWake-SPC`](glossary.md#term-qwake-spc) обозначает возможную последующую
исследовательскую линию, в которой квалифицированные события коррекции
становятся нативными `spikes`, а состояния, перенос ошибок, коммуникация и
обучение получают `spike-native` реализацию.

Перспективная политика может иметь вид

```math
\pi_Q^S:\Phi_t^S\rightarrow(s_t,a_t,k_t,\tau_t,Z_t),
```

где $\tau_t$ задаёт временное окно, а $Z_t$ — нативные `spike`-события.

`QWake-SPC` находится за границей текущей магистерской работы. Этот термин не
обозначает реализованный контроллер, не разрешает эксперимент и не является
условием завершения `A-Min`, `A-Core` или `A-Max`.

## 10. Влияние на текущий план

Критический путь остаётся неизменным:

```text
B1/B2
→ matched profiling
→ EX-IF0
→ passive PC-CATM
→ offline screening
→ predictor
→ exact verification
→ shadow QWake-PC
→ conditional active control
```

В рамках магистерской работы многомасштабность используется как:

1. теоретическая организация уже существующих механизмных и решающих уровней;
2. язык для возможного сравнения блока, слоя и сети;
3. ограниченный разведочный пилот только при наличии уже разрешённых данных;
4. основание для будущей `PhD`-программы, но не новый обязательный эксперимент.

## 11. Архитектурные инварианты

1. `PC-CATM` описывает механизм, но не разрешает действие.
2. Нормы, агрегация, пороги, потери решения и стоимость задаются отдельно для каждого
   масштаба.
3. Локальный `NCZ` или `ECZ` не переносится автоматически на более крупный
   масштаб.
4. `ECZ` не разрешает `local_sweep(block_id)` без отдельной точной проверки.
5. Недостаточность, отсутствие покрытия и небезопасная неопределённость ведут к
   эскалации.
6. Отрицательный результат конкретного контроллера не опровергает `PC-TREF`,
   `PC-CATM` или подтверждённые пассивные наблюдения.
7. Ни один новый масштаб, локальное действие или `spike-native` компонент не
   получает право выполнения из этого документа.

## 12. Проверяемые положения

До отдельной предварительной регистрации следующие положения остаются
гипотезами:

- более дешёвый масштаб может быть достаточен для зарегистрированного решения;
- консервативная эскалация может уменьшить число полных точных проходов;
- дефект композиции может оставаться в зарегистрированном допуске для отдельных
  пар масштабов;
- минимальный достаточный масштаб может зависеть от состояния;
- спайкоподобная организация событий может создать основу для последующего
  `spike-native` расширения.

## 13. Граница утверждений

Документ не устанавливает, что:

- `PC-CATM` инвариантна к масштабу;
- механизмные режимы сохраняются при агрегации;
- координата состояния является биологическим нейроном;
- локальная коррекция сохраняет сетевую конечную точку;
- многомасштабный `QWake-PC` уменьшает полное [время выполнения](glossary.md#term-runtime);
- спайкоподобное управление эквивалентно нейронным `spikes`;
- для `QWake-SPC` существуют эмпирические [доказательные материалы](glossary.md#term-evidence).
