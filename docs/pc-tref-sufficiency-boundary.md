# `PC-TREF-SB`: операционная граница достаточности для `state_inference`

[English version](pc-tref-sufficiency-boundary_EN.md)

## 1. Статус и граница утверждений

Этот документ формализует `design-only` расширение
[PC-TREF](glossary.md#term-pc-tref) после `EX-IF0`. Оно относится к выбору
следующего вычислительного действия во время `state_inference` и не изменяет
B1, B2, их предварительную регистрацию, точные контракты, результаты или
[доказательные материалы](glossary.md#term-evidence).

Обязательный объект магистерской —
[операционная граница достаточности](glossary.md#term-operational-sufficiency-boundary)
для одного следующего действия. Документ не заявляет:

- универсальную границу обучения нейронной сети;
- эквивалентность границы `inference` и границы обновления параметров;
- гладкость, связность или единственную нормаль границы;
- безопасность последовательности активных пропусков;
- разрешение управлять выполнением до отдельных проверок.

## 2. Зарегистрированный контекст решения

Зарегистрированный контекст задаётся кортежем

```math
\Gamma=
(\mathcal X,\mathcal A,a_E,\ell_R,\varepsilon_R,
\delta_{\mathrm{miss}},\mathbf C,\preceq_C,\mathbb P_\Gamma),
```

где:

- $\mathcal X$ — пространство полных состояний эксперимента;
- $\mathcal A$ — фиксированное пространство вычислительных действий;
- $a_E\in\mathcal A$ — зарегистрированное точное эталонное действие;
- $\ell_R(x,a)$ — `task-relative` потеря действия;
- $\varepsilon_R\geq0$ — допустимый `one-step` `regret`;
- $\delta_{\mathrm{miss}}\in[0,1]$ — допустимая вероятность опасного решения политики;
- $\mathbf C(x,a)$ — полный [вектор стоимости](glossary.md#term-cost-vector);
- $\preceq_C$ — предварительно зарегистрированное отношение доминирования по стоимости;
- $\mathbb P_\Gamma$ — распределение состояний внутри зарегистрированного `scope`.

Состояние $x$ включает как минимум [набор данных](glossary.md#term-dataset), пример, `model_seed`, параметры,
скрытое состояние, номер прохода, метод и контекст стоимости. Параметры
фиксируются внутри одного контрфактического сравнения.

## 3. Действия и `one-step` `regret`

Для основной линии:

```math
\mathcal A=
\{a_{\mathrm{stop}},a_{\mathrm{local}},
a_{\mathrm{exact}},a_{\mathrm{fallback}}\}.
```

Операционный `regret` относительно точного эталона:

```math
r_\Gamma(x,a)
=
[\ell_R(x,a)-\ell_R(x,a_E)]_+,
\qquad [z]_+=\max(0,z).
```

Если требуемый ответ является структурным объектом, допускается заранее
зарегистрированный неотрицательный `defect`:

```math
r_\Gamma(x,a)=D_R(Y(x,a),Y(x,a_E)),
\qquad D_R(y,y)=0.
```

Из условия $D_R(y,y)=0$ вместе с первой семантикой следует, что
$a_E\in\mathcal A_\varepsilon(x)$ и допустимое множество непусто. В обоих случаях $r_\Gamma$ является зарегистрированной `exact-reference`
величиной. Она совпадает с классическим `decision regret` только если
$a_E$ реализует зарегистрированное $V^*(x)$ в рассматриваемом пространстве
действий. В противном случае утверждения обязаны использовать формулировку
`exact-reference regret` или `exact-reference defect`, а не глобальную
оптимальность.

Для подтверждающего анализа выбирается одна семантика. Действие допустимо, если

```math
r_\Gamma(x,a)\leq\varepsilon_R.
```

Множество допустимых действий:

```math
\mathcal A_\varepsilon(x)
=
\{a\in\mathcal A:r_\Gamma(x,a)\leq\varepsilon_R\}.
```

## 4. Требуемая эквивалентность

Бинарный класс допустимости `stop` определяется как

```math
q_{\mathrm{stop}}(x)
=
\mathbf 1\{a_{\mathrm{stop}}\in\mathcal A_\varepsilon(x)\}.
```

Он является минимальным требуемым ответом для основной границы этого документа.
Для более сильного `safety`-утверждения вводится отношение равенства полных
множеств допустимых действий:

```math
x\equiv_S y
\Longleftrightarrow
\mathcal A_\varepsilon(x)=\mathcal A_\varepsilon(y).
```

Равенство множеств рефлексивно, симметрично и транзитивно, поэтому
$\equiv_S$ является отношением эквивалентности. Оно допускает `ties` и несколько
безопасных действий. Непустое пересечение допустимых множеств само по себе не
называется эквивалентностью.

Полная [требуемая эквивалентность](glossary.md#term-required-equivalence)
задаётся отдельно зарегистрированной `decision-class` картой $q_R$. Для
`Pareto`-утверждения строгая часть отношения стоимости определяется как

```math
c'\prec_C c
\Longleftrightarrow
c'\preceq_C c\land\neg(c\preceq_C c'),
```

а множество недоминируемых безопасных действий задаётся как

```math
\mathcal P_C(x)
=
\left\{
a\in\mathcal A_\varepsilon(x):
\nexists a'\in\mathcal A_\varepsilon(x),
\mathbf C(x,a')\prec_C\mathbf C(x,a)
\right\}.
```

Допустимые специализации $q_R$ зависят от утверждения:

- для бинарного `stop-safety` утверждения: $q_R(x)=q_{\mathrm{stop}}(x)$;
- для полного `safety`-утверждения: $q_R(x)=\mathcal A_\varepsilon(x)$;
- для утверждения о `Pareto`-множестве:
  $q_R(x)=(\mathcal A_\varepsilon(x),\mathcal P_C(x))$;
- для утверждения об исполненном действии: $q_R(x)$ включает результат заранее
  зарегистрированного `tie-break`.

Во всех случаях

```math
x\equiv_R y
\Longleftrightarrow
q_R(x)=q_R(y).
```

Предсказание знака основной границы устанавливает информацию только о
$q_{\mathrm{stop}}$. Оно само по себе не доказывает достаточность представления
для восстановления полного допустимого множества или выбора между `local`,
`exact` и `fallback` по стоимости.

## 5. Диагностическое представление

Для диагностического бюджета $b$ задаётся отображение

```math
\phi_b:\mathcal X\rightarrow\mathcal Z_b,
```

содержащее только информацию, доступную до точного действия. Для буквального
`quotient` используется зарегистрированная `partition map`

```math
q_b:\mathcal Z_b\rightarrow\mathcal K_b,
```

и отношение

```math
x\equiv_b y
\Longleftrightarrow
q_b(\phi_b(x))=q_b(\phi_b(y)).
```

Пороговая близость

```math
d_b(\phi_b(x),\phi_b(y))\leq\tau_b
```

остаётся `proximity relation` и не создаёт `quotient` без отдельной
`partition map` или доказательства транзитивности.

## 6. Достаточность и два разных дефекта

Представление достаточно для зарегистрированного решения, если

```math
x\equiv_b y\Longrightarrow x\equiv_R y.
```

Точный [дефект относительной к задаче эквивалентности](glossary.md#term-task-relative-equivalence-defect):

```math
\mathfrak D_{b\rightarrow R}=E_b\setminus E_R.
```

Он содержит пары состояний, объединённые диагностическим классом, но требующие
разных множеств допустимых действий.

Для конкретной политики $\pi_b$ отдельно определяется `safety`-дефект:

```math
\mathfrak D_{\mathrm{safe}}(\pi_b)
=
\{x:r_\Gamma(x,\pi_b(\phi_b(x)))>\varepsilon_R\}.
```

Первый объект является множеством пар состояний. Второй является множеством
опасных решений политики. Они не взаимозаменяемы.

## 7. Агрегатные режимы `NCZ`/`ECZ` и отдельный `TNZ`

Пусть полный наблюдатель выдаёт заранее зафиксированные канонические каналы

```math
C(x)=(C_1(x),\ldots,C_m(x)),
```

а зарегистрированный оператор агрегации задаёт $u(x)=A(C(x))$ и удовлетворяет
$A(0)=0$. Точные агрегатные режимы:

```math
\mathrm{NCZ}^0=\{x:C_i(x)=0\ \forall i\},
```

```math
\mathrm{ECZ}^0=
\{x:u(x)=0\land\exists i:C_i(x)\neq0\},
```

```math
\mathrm{active\_non\_ecz}^0=\{x:u(x)\neq0\}.
```

Эти три режима образуют попарно непересекающееся разбиение зарегистрированного
агрегатного `scope`. Малый или нулевой `resultant` не различает `NCZ` и `ECZ`.

`TNZ` не является третьим агрегатным дополнением. Он определяется отдельно для
оператора переноса как ненулевой источник ошибки в ядре сопряжённого якобиана:

```math
\mathrm{TNZ}^0
=
\{x:e_{l+1}(x)\neq0\land
\widetilde J_{h,l+1}^{*}e_{l+1}(x)=0\}.
```

Поэтому одно состояние может одновременно иметь определённый агрегатный режим
и отдельный статус переноса. Численные окрестности задаются контрактом
[нуля, маскированного точностью](glossary.md#term-precision-masked-zero) и не
являются разрешением на действие.

## 8. Односторонние сертификаты и `abstention`

Дешёвый `NCZ`-сертификат требует верхних границ $U_i(x)$:

```math
\|C_i(x)\|\leq U_i(x),
\qquad U_i(x)\leq\tau_i\ \forall i.
```

Отсутствие обнаруженной активности недостаточно.

Дешёвый `ECZ`-сертификат требует одновременно:

```math
U_u(x)\leq\tau_u,
\qquad L_{\mathrm{act}}(x)>\tau_{\mathrm{act}},
```

и зарегистрированных `geometry guards`. Если `bounds` не дают одностороннего
решения, результат равен `abstained`. При пересечении доверительных `bounds`
сертификатор также обязан воздержаться.

Любой сертификат сохраняет `action_permission=none` до отдельного
предварительно зарегистрированного предиктора, точной проверки, `regret`-барьера
и теневого режима.

## 9. Накопленный диагностический спрос

Для слоя $l$ вводится канонический `pre-action` сигнал

```math
c_l(t)\in\mathbb R^{d_l}.
```

Он называется вектором диагностического спроса на дополнительный `inference`, а
не вектором потребности параметрического обучения.

Подписанный накопитель:

```math
p_l(t+1)=\Lambda_l^p p_l(t)+c_l(t).
```

Некомпенсируемый накопитель активности:

```math
e_l(t+1)=\Lambda_l^e e_l(t)+|c_l(t)|.
```

Операторы памяти $\Lambda_l^p$ и $\Lambda_l^e$, их спектральные радиусы,
единицы шага, нормы и правила сброса фиксируются до анализа. Большое $e_l$ при
малом $p_l$ интерпретируется только относительно зарегистрированных норм,
масштабов и порогов. Оно является признаком компенсации или осцилляции, но не
самостоятельной меткой `ECZ` и не разрешением на локальное действие.

## 10. `Oracle` `one-step` граница

Для `stop` определяется недоступный `online` `oracle` `regret`:

```math
r_{\mathrm{skip}}^*(x)=r_\Gamma(x,a_{\mathrm{stop}}).
```

[Oracle-запас достаточности](glossary.md#term-oracle-sufficiency-margin):

```math
M^*(x)=\varepsilon_R-r_{\mathrm{skip}}^*(x).
```

Тогда:

```math
\mathcal S^*=\{x:M^*(x)\geq0\},
```

```math
\mathcal U^*=\{x:M^*(x)<0\},
```

```math
\mathcal B^*=\{x:M^*(x)=0\}.
```

Бинарная `oracle`-метка основной задачи:

```math
y_{\mathrm{stop}}^*(x)
=
\mathbf 1\{M^*(x)\geq0\}
=
q_{\mathrm{stop}}(x).
```

$\mathcal B^*$ по соглашению называется `one-step` операционной границей, но
формально является пороговым `level set`. Она не обязана совпадать с
топологической границей $\partial\mathcal S^*$. Такое совпадение требует как
минимум зарегистрированной топологии, достаточной регулярности $M^*$ и локальной
смены знака. Из определения также не следуют гладкость, связность, единственная
нормаль или переносимость за пределы $\Gamma$.

## 11. `Pre-action` оценка границы

[Оцениватель границы достаточности](glossary.md#term-sufficiency-boundary-estimator)
использует только `pre-action` информацию:

```math
\widehat M_b(x)=g_b(\phi_b(x)).
```

$M^*$ является `post-action` `oracle` `label`, а $\widehat M_b$ — его диагностическая
оценка. Их разделение запрещает циклическое определение.

Предиктор возвращает интервал

```math
I_b(x)=[L_b(x),U_b(x)].
```

Интерпретация при зарегистрированной процедуре построения интервала:

- $L_b(x)\geq0$: [кандидат](glossary.md#term-candidate) классифицируется как безопасный;
- $U_b(x)<0$: кандидат классифицируется как небезопасный;
- $L_b(x)<0\leq U_b(x)$: `abstained` или `fallback_exact`.

Эти классификации поддерживают `safety`-утверждение только после отдельной
валидации покрытия интервала на данных, не использованных для его построения.
Точечное значение $\widehat M_b>0$ само по себе не является
`safety`-сертификатом.

## 12. `First-order` горизонт

Для фиксированного диагностического бюджета $b$ изменение оценённого запаса:

```math
\Delta\widehat M_b(t)
=
\widehat M_b(t)-\widehat M_b(t-1).
```

Скорость расходования запаса:

```math
V_b^M(t)=\max(0,-\Delta\widehat M_b(t)).
```

[Прогнозируемый горизонт достаточности](glossary.md#term-predicted-sufficiency-horizon):

```math
\widehat H_b^{(1)}(t)
=
\frac{\max(0,\widehat M_b(t))}
{V_b^M(t)+\epsilon_H},
\qquad \epsilon_H>0.
```

Численная константа $\epsilon_H$, единица шага и максимальный горизонт
фиксируются до анализа. Консервативная версия использует нижнюю границу $L_b$.
Это локальная `first-order` оценка при сохранении текущего темпа, а не
гарантированное число безопасных пропусков.

Индекс слоя $l$ разрешается вместо $b$ только если отдельно определены
`layer-specific` действие и `oracle`-метка $M_l^*$. Использование признаков слоя
для оценки глобального $M^*$ само по себе не создаёт границу слоя.

## 13. Условное геометрическое расширение

Геометрическая интерпретация разрешается только при дополнительных условиях:

- существует локально дифференцируемый `surrogate` $h_l(z)$;
- зарегистрировано, оценивает ли он глобальный $M^*$ или отдельно определённый $M_l^*$;
- $h_l(z)=0$ задаёт только соответствующий `surrogate` `boundary`;
- $\nabla h_l(z)\neq0$ в рассматриваемой точке;
- зарегистрирована положительно определённая метрика $G_l$.

Градиент относительно метрики:

```math
\operatorname{grad}_{G_l}h_l=G_l^{-1}\nabla h_l.
```

Нормализованное направление:

```math
n_l^G
=
\frac{\operatorname{grad}_{G_l}h_l}
{\|\operatorname{grad}_{G_l}h_l\|_{G_l}}.
```

Нормальная скорость:

```math
q_l=\langle n_l^G,\dot z_l\rangle_{G_l}.
```

Касательность к `surrogate` `level` `set` означает только малую локальную
чувствительность выбранного `surrogate` первого порядка. Она не доказывает
сохранение `required-equivalence` `class`. В негладком случае требуются `normal` `cone`,
`subgradient` или `directional` `derivative`; это не входит в обязательный `claim`.

## 14. `ECZ`-защищённая нормальная активность

При заранее зафиксированной декомпозиции

```math
\dot z_l=\sum_i\dot z_{l,i}
```

можно сравнивать

```math
q_l^{\mathrm{net}}
=
\left\langle n_l^G,\sum_i\dot z_{l,i}\right\rangle_{G_l},
```

```math
q_l^{\mathrm{abs}}
=
\sum_i
\left|\langle n_l^G,\dot z_{l,i}\rangle_{G_l}\right|.
```

Малое $q_l^{\mathrm{net}}$ при большом $q_l^{\mathrm{abs}}$ обозначает возможную
нормальную компенсацию. $q_l^{\mathrm{abs}}$ зависит от канонической
декомпозиции и не является инвариантным к произвольной замене базиса.

## 15. `Safety-first` выбор и `cost-sufficiency` `frontier`

Сначала формируется множество действий, прошедших верхнюю границу `regret`:

```math
\widehat{\mathcal A}_{\mathrm{safe}}(x)
=
\{a:\widehat r^{\,\mathrm{upper}}(x,a)\leq\varepsilon_R\}.
```

Только затем применяется [Pareto-допустимость](glossary.md#term-pareto-admissibility)
по полному вектору стоимости. Стоимость не превращает небезопасное действие в
допустимое. В основной `stop-boundary` ветви такой верхний `regret`-`bound`
строится только для $a_{\mathrm{stop}}$; `local` входит в безопасное множество
лишь после отдельной `action-specific` точной проверки или оценивателя.

Для бюджета $b$ допустимость определяется как

```math
\Pr_{\mathbb P_\Gamma}
[r_\Gamma(x,\pi_b(\phi_b(x)))>\varepsilon_R]
\leq\delta_{\mathrm{miss}}.
```

Итоговый объект — множество `Pareto`-минимальных диагностических представлений
среди прошедших `safety`-ограничение. Единственная глобально минимальная
репрезентация не заявляется.

## 16. Эмпирические гипотезы и барьеры

Будущий отдельный протокол проверяет:

- превосходит ли $\widehat M_b$ простые нормы по `safety`–`cost` `trade-off`;
- даёт ли $\widehat H^{(1)}$ дополнительную `out-of-sample` информацию;
- снижает ли `ECZ`-защита `false-safe` `rate`;
- предсказывает ли пространственная структура допустимость локального действия;
- остаётся ли диагностика дешевле предотвращённого exact `sweep`;
- воспроизводится ли результат на уровне независимых `model_seed`.

Нулевое наблюдаемое число `dangerous_miss` должно сопровождаться заранее
зарегистрированной верхней доверительной границей для вероятности ошибки. Оно
не является доказательством универсальной безопасности.

Обязательная последовательность:

```text
EX-IF0
→ policy-neutral oracle labels
→ passive boundary features
→ offline validation
→ predictor preregistration
→ shadow mode
→ conditional active mode
```

`B1`/`B2`, `EX-IF0`, точная реализация, `model_seed` как независимая единица,
закрытая тестовая выборка и `controls_execution=false` сохраняются. `One-step`
результат не выдаётся за `trajectory-level` безопасность.
