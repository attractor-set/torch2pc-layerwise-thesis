# Воспроизводимость

[English version](reproducibility_EN.md)

## Фиксируемые объекты

| Объект | Механизм |
|---|---|
| Код проекта | Git commit |
| Torch2PC | полный SHA и detached checkout |
| Контейнер | tag, image ID, base digest |
| Python | lock-файлы и `pip freeze` |
| Данные | локальный manifest и SHA-256 |
| Разбиения | сохраненные NPZ и SHA-256 |
| Случайность | split/model/loader/corruption seed |
| Исполнение | dtype, device, workers, threads, batch size |
| Попытка | уникальный `run_id` |
| Результаты | resolved config, metrics, environment, manifest |

## Детерминизм

`torch.use_deterministic_algorithms` включается в строгом режиме. Если операция
не поддерживает детерминизм, запуск должен завершиться ошибкой либо получить
отдельную документированную конфигурацию. Предупреждение без остановки не
считается достаточным для основного анализа.

## Повторяемость и воспроизводимость

- повторяемость: тот же узел, image, commit и config;
- воспроизводимость: другой совместимый узел с теми же артефактами;
- статистическая устойчивость: независимые model seeds.

Совпадение отдельных floating-point значений между разными GPU не требуется;
сравниваются заранее заданные допуски и статистические выводы.

## Lock-файл окружения

`environment-lock.json` связывает контрольные и экспериментальные запуски с:

- SHA-256 исходного кода и конфигураций;
- неизменяемыми Docker image ID и RepoDigest;
- списком пакетов внутри контейнера;
- полным commit Torch2PC;
- сведениями о хосте, драйвере и ROCm-инструментах.

После изменения исходного кода или конфигурации lock-файл создается заново, а
C0/C1 повторяются. Контрольные результаты содержат SHA-256 lock-файла, поэтому
артефакт другой среды не принимается автоматически.

## Проверяемость метрик

Каждый завершенный запуск сохраняет validation predictions, а final также test
predictions вместе с исходными индексами объектов, истинными метками,
предсказаниями и вероятностями. Это позволяет повторно вычислить агрегированные
метрики без повторного запуска модели.

Публичный пакет включает код, конфигурации, библиографию, агрегированные
результаты, рисунки и таблицы, но не PDF публикаций, исходные датасеты и тяжелые
checkpoints.
