# Этап 3B `SI-MA0`: подтверждающая агрегация и доказательные материалы

[English version](stage3b-si-ma0-confirmatory-aggregation_EN.md)

## Статус

Этот документ описывает отдельный этап анализа и публикации после завершения
десяти неизменяемых подтверждающих ячеек `SI-MA0`.

[Выполнение](../glossary.md#term-execution) уже заморожено на следующих значениях:

```text
execution source commit = 03016e68ecc7a850da7148d676f47acfb07cc99e
image revision          = 03016e68ecc7a850da7148d676f47acfb07cc99e
contract id             = stage3b-si-ma0-v2
model seeds             = 0..9
validation batch ids    = 0,1,2
```

Код агрегации создаётся отдельным коммитом реализации. Сформированные
подтверждающие [доказательные материалы](../glossary.md#term-evidence)
фиксируются последующим коммитом доказательных материалов. Исходные рабочие
попытки не изменяются и не включаются в коммит реализации.

## Принципы

Пакет следует принципам проекта:

- сначала фиксируется протокол: пороги, области, случайные начальные значения и
  оцениваемые величины остаются замороженными;
- [независимая статистическая единица](../glossary.md#term-statistical-unit) —
  отдельно обученная модель, заданная полем `model_seed`;
- отрицательные результаты сохраняются;
- научно отрицательный результат не является основанием для замены запуска или
  повторной настройки;
- исходный код и образ выполнения отделены от исходного кода агрегации;
- исходные записи объединяются без фильтрации после просмотра результата;
- русский отчёт является основным, английский создаётся парно;
- вывод ограничивается зарегистрированными контрольными точками, пакетами данных
  и вычислительной линией ROCm.

## Входы

Агрегация принимает:

1. десять каталогов
   `results/stage-3/si-ma0/working/confirmatory/seed-<n>/primary-03016e68ecc7`;
2. внутренний `SHA256SUMS` каждой ячейки;
3. внешний реестр десяти попыток и архивов по случайным начальным значениям;
4. замороженную инвентаризацию каждой
   [контрольной точки](../glossary.md#term-checkpoint);
5. общий архив совокупности запусков;
6. замороженный `STAGE3B-SI-MA0-CONTRACT.json`;
7. запечатанные предварительные проверки A1;
8. запечатанную сводку временных измерений ROCm `OBS-OH0` для `joint-VJP`.

`OBS-OH0` используется только как описательный контекст. Его допустимый уровень
накладных расходов не заменяет пятипроцентный порог `COST-MA0`, поскольку
оцениваемые величины и пути выполнения различаются.

## Проверки полноты

До создания доказательных материалов агрегатор проверяет:

- наличие ровно случайных начальных значений `0..9`;
- одинаковые коммит исходного кода выполнения и редакцию образа;
- `contract_id = stage3b-si-ma0-v2`;
- три валидационных пакета данных для каждого случайного начального значения;
- принятие решений по всем ячейкам;
- отсутствие доступа к тестовой выборке;
- успешную проверку внутренних манифестов SHA256;
- успешную проверку внешних архивов и их хэшей;
- успешную проверку контрольных точек и их инвентаризации;
- обязательные количества `3000/600/3600/150/7500/52500/70`;
- одинаковые схемы CSV между случайными начальными значениями;
- отсутствие пропущенных или повторяющихся ключей временных измерений;
- независимое воспроизведение сохранённых сумм областей и остатка учёта времени.

При любой неполноте выходной каталог удаляется целиком, а доказательные материалы
не создаются.

## Основные оцениваемые величины

Для поля `model_seed`, обозначенного `m`, и области `r`:

```text
s[m,r] = sum(region_device_time[m,r]) /
         sum(state_inference_total_device_time[m])
```

Суммирование сначала выполняется внутри одного случайного начального значения по
пакету данных, повторению и измеряемому шагу. Между десятью случайными начальными
значениями публикуются:

- все значения уровня случайного начального значения;
- медиана;
- Q1, Q3 и межквартильный размах;
- среднее;
- 95% доверительный интервал процентильного бутстрэпа для медианы;
- дополнительный 95% бутстрэп-интервал для среднего;
- `10000` повторов;
- случайное начальное значение бутстрэпа `20260715`;
- только `model_seed` как единица повторной выборки.

Неатрибутированный остаток публикуется отдельно и не перераспределяется между
семью замороженными областями.

## Глобальные проверки

Глобальное решение вычисляется только после полной агрегации:

```text
si_ma0_passed =
    prerequisites_verified
    and REC-MA0
    and OBS-MA0
    and VER-MA0
    and COST-MA0
    and CMP-MA0
```

`COST-MA0` независимо пересчитывается по 7500 измеряемым шагам и 150 агрегатам
уровня повторения с замороженным порогом `rho <= 0.05`, минимальной долей шагов
`0.99` и долей повторений `1.0`.

## Выходы

Доказательные материалы создаются в:

```text
results/stage-3/si-ma0/confirmatory/
```

Пакет содержит объединённые обязательные исходные записи, сводки по случайным
начальным значениям, основные доли вычислительной стоимости, бутстрэп-статистики,
статистики остатка, манифест исходных попыток, описательный контекст `OBS-OH0`,
глобальную сводку и решение, парные русский и английский отчёты и общий
`SHA256SUMS`.

## Граница интерпретации

Близость остатка учёта времени к ранее измеренным накладным расходам наблюдателя
ROCm `joint-VJP` может быть опубликована как описательное согласование масштаба.
Она не доказывает идентичность величин и не меняет формальный `COST-MA0`.

При полном наборе доказательных материалов и хотя бы одной неуспешной проверке
итоговое состояние — `fail`, а не `inconclusive`. Такой результат не открывает
интерпретацию `NCZ/ECZ/TNZ` и последующие проверки B1/B2.
