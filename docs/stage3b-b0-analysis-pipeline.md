# Этап 3B B0: конвейер статистического и инженерного анализа

[English version](stage3b-b0-analysis-pipeline_EN.md)

## Назначение

`stage3b_b0_analysis.py` реализует анализ опубликованного `sealed-v1` только
для чтения. Интерфейс командной строки:

```bash
python scripts/analyze_stage3b_b0.py \
  --evidence-root results/stage-3/profiling/b0/sealed-v1 \
  --output-root results/stage-3/profiling/b0/analysis-v1 \
  --source-commit <merged-analysis-implementation-commit> \
  --generated-at-utc <frozen-UTC-timestamp>
```

## Проверки

До расчётов конвейер проверяет:

- точное соответствие `SHA256SUMS`;
- отсутствие пропущенных или неожиданных файлов;
- границу утверждений опубликованной фиксации;
- 96 экспериментальных ячеек, 480 строк областей измерения, 48 пар и 32 строки
  конфигураций;
- полную факторную матрицу и три случайных начальных значения модели на
  конфигурацию;
- флаги целостности и отсутствие неконечных значений.

После записи результатов закреплённый вход проверяется повторно. Запись
результатов внутри `sealed-v1` запрещена.

## Аналитические ограничения

- [независимая статистическая единица](glossary.md#term-statistical-unit): `model_seed`;
- независимое `n=3` на конфигурацию;
- p-значения не используются для утверждений о превосходстве;
- матрица конфигураций анализируется описательно;
- доли областей нормализуются внутри суммы медиан по областям;
- модели масштабирования не включают взаимодействия;
- полный этап 3B остаётся незавершённым.

## Разделение программного обеспечения и доказательных материалов

Рекомендуются два коммита:

1. коммит программного обеспечения с модулем, интерфейсом командной строки,
   тестами и зафиксированным контрактом анализа;
2. после слияния — отдельный коммит доказательных материалов со
   сгенерированным `analysis-v1`.

Это позволяет метаданным фиксировать слитый коммит реализации анализа и
проверять точное дерево опубликованного коммита до публикации.
