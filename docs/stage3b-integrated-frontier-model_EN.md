# Stage 3B: integrated frontier model

[Русская версия](stage3b-integrated-frontier-model.md)

**Status:** design freeze after `ADR-039`; [execution](glossary_EN.md#term-execution), oracle-label generation, feature collection, and control remain closed.

## 1. Purpose

This document introduces one state model for FixedPred sufficiency, observation
cost, and future [QWake-PC](glossary_EN.md#term-qwake-pc) orchestration. It does not change FixedPred dynamics,
the exact `stage2_baseline` reference, or `EX-IF0`. It only separates:

1. where canonical computation has reached;
2. how much pre-action information has been acquired;
3. which action is proposed relative to the current frontier.

`ADR-039` remains historically and normatively unchanged. Its
`DONE / UNKNOWN / SWEEP` alphabet remains the shadow admission and uncertainty
semantics. The alphabet below is the more precise action alphabet.

## 2. Architectural-layer separation

Roles do not collapse:

```text
PC-TREF  = sufficiency, admissible regret, and action-admission semantics
PC-CATM  = mechanism model and source of interpretable evidence
QWake-PC = orchestration of registered frontier transitions
executor = canonical stage2_baseline trajectory
```

PC-TREF determines whether the current frontier may be accepted relative to the
registered final answer. PC-CATM may provide mechanism-derived features but
cannot authorize an action. QWake-PC may only choose among registered
transitions and remains `controls_execution=false` before separate admission.

A negative result for one orchestration does not invalidate PC-TREF or PC-CATM.

## 3. Integrated frontier

For the mandatory temporal path, a frontier state is

```math
\mathcal F_{t,i}=
\left(S_t, R_{A_i}, B^{\mathrm{compute}}_{\mathrm{rem}},
B^{\mathrm{diag}}_{\mathrm{rem}}, P\right),
```

where:

- $S_t$ is the immutable snapshot after $t$ canonical sweeps;
- $R_{A_i}$ is the cumulative pre-action observation representation;
- $B^{\mathrm{compute}}_{\mathrm{rem}}$ is the remaining compute budget;
- $B^{\mathrm{diag}}_{\mathrm{rem}}$ is the remaining diagnostic budget;
- $P$ is model, data, RNG, graph, implementation, and provenance identity.

A future spatial extension may add a preregistered nested aggregate $B_k$ to the
frontier identity. That extension is outside the mandatory temporal core and
does not open recursive-aggregate execution.

The frontier is monotone: a transition cannot undo a completed sweep, remove an
acquired level, or change snapshot provenance.

## 4. The `A0 / A1 / A2` observation axis and `O`

### `A0` — structural observation

A0 contains only already available structural fields: snapshot, method, sweep,
layer, and block identifiers; registered topology; shapes; dtype; budgets; and
membership in preregistered aggregates. Tensor values are not read.

### `A1` — cheap device-side reductions

A1 adds a finite registered set of reductions over tensors that are already
available. [Device time](glossary_EN.md#term-device-time), synchronization, transferred bytes, and temporary
memory are measured rather than treated as free.

### `A2` — local reductions on a deterministic subsample

A2 adds local reductions over indices derived only from pre-action identity and
a frozen sampling key. Subsamples form the nested family

```math
S_0\subseteq S_1\subseteq\cdots\subseteq S_q.
```

The key, sizes, and order are frozen before labels are generated.

### `O` — post-action oracle only

O is generated by the full canonical suffix on a separately frozen subset:

```math
U_O\subseteq U_{A2}\subseteq U_{A1}=U_{A0}.
```

O is not an available-frontier axis, is not part of a deployable
representation, and is never passed to a shadow controller.

Cumulative representations are fixed:

```text
R_A0 = A0
R_A1 = A0 + A1
R_A2 = A0 + A1 + A2
O    = post-action labels only
```

## 5. Action alphabet

The integrated model has exactly three top-level actions.

### `ACCEPT_FRONTIER`

Accept the current compute frontier as sufficient for the registered final
answer. This action requires positive PC-TREF safety and risk admission. Cost,
NCZ, ECZ, a small residual, or component completion cannot by themselves
authorize ACCEPT_FRONTIER.

### `ADVANCE_FRONTIER`

Perform exactly one registered adjacent transition:

1. acquire the next observation level while keeping $S_t$ unchanged; or
2. execute one next canonical sweep and open $S_{t+1}$ at base level A0.

The transition kind, target identity, and complete [cost vector](glossary_EN.md#term-cost-vector) are retained
explicitly. A transition cannot skip an unregistered level, execute multiple
sweeps, or select its target from a post-action label.

### `COMPLETE_SUFFIX`

Execute the full remaining canonical `stage2_baseline` suffix through the
registered reference horizon. This is the only fail-closed [fallback](glossary_EN.md#term-fallback) when
admission, the diagnostic budget, the transition registry, or state integrity
is exhausted.

In this package all three actions are shadow proposals only. The actual
trajectory remains the full canonical suffix.

## 6. Mapping from `D / U / S`

ADR-039 is not rewritten. The compatible mapping is:

| `ADR-039` | Integrated interpretation |
|---|---|
| `DONE` | ACCEPT_FRONTIER [candidate](glossary_EN.md#term-candidate) requiring positive admission |
| `UNKNOWN` | epistemic state, not an action by itself |
| `UNKNOWN` with an admissible next level | `ADVANCE_FRONTIER(observation)` |
| `SWEEP` as one next canonical sweep | `ADVANCE_FRONTIER(compute)` |
| unresolved uncertainty or contract violation | `COMPLETE_SUFFIX` |

UNKNOWN therefore no longer overloads acquisition, and SWEEP no longer means
both one sweep and the full fallback suffix.

## 7. Mandatory-core transition graph

For each $t$, the registered graph is:

```text
F(t, A0) --acquire_A1--> F(t, A1)
F(t, A1) --acquire_A2--> F(t, A2)
F(t, Ai) --one_sweep----> F(t+1, A0)
F(t, Ai) --accept-------> terminal_shadow_accept
F(t, Ai) --full_suffix--> terminal_canonical_reference
```

A missing, unavailable, or unaffordable adjacent edge is not replaced by a
hidden alternative. COMPLETE_SUFFIX applies instead.

The first mandatory shadow cascade remains deterministic:

```text
A0 -> A1 -> A2 -> ADVANCE_FRONTIER(compute) | COMPLETE_SUFFIX
```

Greedy ordering, learned analytic selection, active spatial locality, and
spike-like orchestration are outside the mandatory core.

## 8. Transition-cost accounting

Cost belongs to a frontier edge rather than only to a terminal decision:

```math
\Delta\mathbf C(e)=
\left(
C_{\mathrm{host}}, C_{\mathrm{device}}, N_{\mathrm{sync}},
B_{\mathrm{d2h}}, M_{\mathrm{temp}}, B_{\mathrm{trace}}
\right).
```

Required fields are:

```text
host_elapsed_ns
device_elapsed_ns
synchronization_events
device_to_host_bytes
temporary_peak_bytes
persistent_trace_bytes
acquisition_count
```

Marginal and cumulative costs are retained for A0, A1, A2, one canonical
sweep, and the full suffix. O-generation cost is separate research-labeling
cost and is not included in deployable diagnostic cost.

Savings are considered only after safety admission:

```text
safety -> safe coverage -> UNKNOWN burden -> full cost vector -> stability
```

## 9. Non-interference and temporal boundary

Observation acquisition must be read-only and cannot change state, parameters,
buffers, gradients, RNG, the autograd path, or canonical branch selection. Any
mutation is an architectural violation.

```text
observer_state_mutations=0
observer_parameter_mutations=0
observer_buffer_mutations=0
observer_rng_mutations=0
observer_autograd_path_changes=0
observer_interference_events=0
post_action_feature_leakage=0
```

Only current-frontier fields are available before action. $M^*$, $t^*$, the
terminal [endpoint](glossary_EN.md#term-endpoint), realized next-sweep value, and the oracle-optimal transition
are post-action information.
