# Жизненный цикл эксперимента

[English version](README_EN.md)

`registry.csv` является append-only журналом событий. Каждая попытка имеет
уникальный `run_id` и обычно две записи: `running`, затем `completed` или
`failed`.

Последняя запись для `run_id` определяет состояние конкретной попытки. Один
`experiment_id` может иметь несколько независимых попыток; предыдущие каталоги
не перезаписываются.

- `planned/` - предварительно описанные серии;
- `completed/` - краткие резюме наблюдений после анализа;
- `failed/` - разбор технических ошибок;
- `registry.csv` - машиночитаемый журнал;
- `results/runs/` - локальные артефакты попыток.

Статус `completed` означает завершение процедуры, а не истинность гипотезы.
