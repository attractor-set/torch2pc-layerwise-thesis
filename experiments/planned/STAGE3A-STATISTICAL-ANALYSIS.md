# Stage 3A — план статистического анализа

## 1. Область анализа

- Набор данных: FashionMNIST.
- Архитектура: `lenet_classic`.
- Контрольная точка: `final`.
- Сиды модели: `0–9`.
- Независимая статистическая единица: отдельно обученный `model_seed`.
- Число независимых наблюдений: `n = 10`.

Отдельные пакеты данных, параметры и пары слоёв не рассматриваются как
независимые экспериментальные повторения.

## 2. Исследуемые методы

Основные сравнения:

- FixedPred относительно BP;
- Strict относительно BP.

Exact относительно BP используется как численный контроль корректности
реализации и не входит в основные проверяемые гипотезы.

## 3. Градиентные метрики

Анализируются:

- cosine similarity;
- relative L2;
- norm ratio;
- sign agreement.

Исходные наблюдения агрегируются внутри каждого:

- `model_seed`;
- метода;
- слоя;
- типа параметра;
- метрики.

Для подтверждающего анализа формируется одно итоговое значение на
`model_seed × method × layer × metric`.

## 4. Метрики представлений

Анализируются:

- corresponding-layer linear CKA;
- corresponding-layer RSA;
- cross-layer CKA.

Для corresponding-layer метрик формируется одно значение на
`model_seed × method × layer × metric`.

Cross-layer CKA используется для описания структуры соответствия между
слоями и анализируется отдельно от corresponding-layer гипотез.

## 5. Описательная статистика

Для каждой комбинации метода, слоя и метрики публикуются:

- число валидных сидов;
- среднее;
- медиана;
- стандартное отклонение;
- минимум и максимум;
- 95% доверительный интервал;
- число неопределённых или пропущенных значений.

Доверительные интервалы рассчитываются на уровне независимых сидов.

## 6. Проверка гипотез

При `n = 10` используется точный парный sign-flip permutation test.

Для каждого контраста перебираются все `2^10 = 1024` возможных перестановки
знаков парных различий.

Двустороннее p-значение рассчитывается относительно абсолютного значения
наблюдаемой средней парной разницы.

## 7. Размер эффекта

Для каждого парного сравнения публикуются:

- средняя парная разница;
- медианная парная разница;
- стандартизованный парный эффект `dz`;
- rank-biserial correlation;
- 95% доверительный интервал средней парной разницы.

Интерпретация результата основывается на величине эффекта и доверительном
интервале, а не только на p-значении.

## 8. Коррекция множественных сравнений

Применяется пошаговая коррекция Holm.

Коррекция выполняется отдельно внутри заранее определённых семейств:

1. FixedPred — градиентные метрики по слоям;
2. Strict — градиентные метрики по слоям;
3. FixedPred — corresponding-layer CKA/RSA по слоям;
4. Strict — corresponding-layer CKA/RSA по слоям.

Семейства и их состав фиксируются до генерации подтверждающих результатов.

## 9. Анализ глубины

Порядок слоёв отображается на нормированную глубину `[0, 1]`.

Для каждого `model_seed` и метода рассчитывается связь между глубиной и
метрикой:

- Spearman correlation;
- линейный наклон как описательная характеристика.

Статистический анализ выполняется над десятью seed-level коэффициентами.
Слои одной сети не считаются независимыми повторениями.

## 10. Exact как численный контроль

Для Exact относительно BP проверяются:

- cosine similarity, близкая к `1`;
- relative L2, близкая к `0`;
- norm ratio, близкий к `1`;
- CKA и RSA, близкие к `1`.

Exact используется для обнаружения ошибок извлечения, агрегации или
сопоставления данных. Для него не формулируется содержательная гипотеза о
превосходстве метода.

## 11. Обработка неопределённых значений

- Неопределённые cosine и RSA сохраняются как отсутствующие значения.
- Значения не заменяются нулями.
- Для каждой строки результата публикуется фактическое `n`.
- Сравнение выполняется только по сидам, где присутствуют обе стороны пары.
- Любое снижение `n` ниже 10 явно отражается в отчёте.

## 12. Выходные артефакты

Статистический анализ должен создать:

- `seed_level_gradient_metrics.csv`;
- `seed_level_representation_metrics.csv`;
- `gradient_statistics.csv`;
- `representation_statistics.csv`;
- `depth_statistics.csv`;
- `exact_numerical_control.csv`;
- `analysis_metadata.json`;
- графики в каталоге `figures/`.

Метаданные должны содержать:

- commit исходного кода;
- SHA-256 входных таблиц;
- дату анализа;
- версию Python и библиотек;
- определения статистических семейств;
- параметры доверительных интервалов и permutation test.

## 13. Критерии завершения

Stage 3A statistical analysis считается завершённым, когда:

- все десять сидов учтены либо пропуски явно документированы;
- Exact прошёл численный контроль;
- сформированы seed-level таблицы;
- рассчитаны эффекты, интервалы и точные p-значения;
- применена коррекция Holm;
- результаты воспроизводятся из опубликованных summary CSV;
- тесты и проверки качества проходят;
- итоговые таблицы и рисунки зафиксированы отдельным evidence commit.

## 14. Правило заморозки

После фиксации этого документа без отдельного обоснования не изменяются:

- статистическая единица;
- правила агрегации;
- набор основных сравнений;
- метрики;
- семейства коррекции;
- методы проверки гипотез;
- критерии завершения.

Любое отклонение оформляется как отдельная поправка с объяснением причины.
