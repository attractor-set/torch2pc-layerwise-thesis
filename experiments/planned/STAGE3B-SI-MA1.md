# Этап 3B `SI-MA1` — калиброванное по наблюдателю замыкание стоимости

[English version](STAGE3B-SI-MA1_EN.md)

## Статус

Этот документ и машинный контракт образуют замороженную предварительную
регистрацию этапа `A9`. Пакет должен быть зафиксирован аннотированным тегом
`stage3b-si-ma1-prereg-v1` до создания исполняемой реализации. Первое [выполнение](../../docs/glossary.md#term-execution) типа `controlled smoke` разрешено только после тега.

Пакет регистрирует новую проверку. Он не заменяет `SI-MA0`, не меняет его
порог и сохраняет его итог:

```text
REC-MA0  = pass
OBS-MA0  = pass
VER-MA0  = pass
COST-MA0 = fail
CMP-MA0  = pass
si_ma0_passed = false
```

Идентификатор контракта:

```text
stage3b-si-ma1-v1
```

## Исследовательский вопрос

Объясняется ли неатрибутированный временной разрыв боевого [вывода состояний](../../docs/glossary.md#term-state-inference) локально
сопоставленной стоимостью наблюдателя так, что односторонняя верхняя
95-процентная доверительная граница избыточного разрыва не превышает
одного процентного пункта?

## Трёхрежимный блок

Для одной контрольной точки, одного пакета данных и одного восстановленного
начального состояния выполняются три режима:

```text
A = baseline_no_observer
B = calibration_observer
C = live_attribution
```

- `A` использует `no_hooks`.
- `B` использует `counters_only` и оценивает стоимость наблюдателя.
- `C` использует тот же `counters_only`, но сохраняет области боевой
  атрибуции.
- `B` и `C` обязаны использовать один исполняемый путь, одинаковые `hooks`,
  `events`, границы областей, запись буферов и синхронизацию.
- Роль `B` или `C` назначается вне [области измерения](../../docs/glossary.md#term-measured-region).
- Сериализация выполняется после измеряемого участка.

Перед каждым режимом заново восстанавливается [контрольная точка](../../docs/glossary.md#term-checkpoint), `RNG snapshot`, начальные `beliefs`, входы и `targets`. Это является новой [проверкой численной эквивалентности](../../docs/glossary.md#term-numerical-equivalence-check),
а не продолжением состояния предыдущего режима.

## Сбалансированный порядок

Один трёхрежимный блок повторяется шесть раз для каждой комбинации
`model_seed × validation_batch_id`.

Зарегистрированы все перестановки:

```text
ABC
BCA
CAB
ACB
CBA
BAC
```

Список циклически сдвигается на

```text
(model_seed + validation_batch_id) mod 6
```

Каждая перестановка встречается ровно один раз. Каждый режим занимает
каждую позицию ровно два раза. Порядок нельзя менять по промежуточному
результату.

Это [сопоставленное профилирование](../../docs/glossary.md#term-matched-profiling). Каждый отдельный [запуск](../../docs/glossary.md#term-run) входит в сбалансированный блок; три изолированных измерения не используются.

## Область эксперимента

Зарегистрированы [набор данных](../../docs/glossary.md#term-dataset) `FashionMNIST` и [архитектура](../../docs/glossary.md#term-architecture) `lenet_classic`.

```text
dataset = FashionMNIST
split = validation
architecture = lenet_classic
method = Strict
checkpoint = final
model_seeds = 0..9
validation_batch_ids = 0,1,2
batch_size = 256
dtype = float32
device = ROCm
eta = 0.05
inference_steps = 20
optimizer_step = disabled
parameter_update = disabled
test_split_access = false
```

[Независимая статистическая единица](../../docs/glossary.md#term-statistical-unit) — отдельно обученная
модель. Поле `model_seed` идентифицирует
[случайное начальное значение модели](../../docs/glossary.md#term-model-seed). Пакет данных, блок порядка,
режим и измеряемый шаг являются вложенными наблюдениями.

## Временной протокол

Для каждой комбинации `model_seed × validation_batch_id`:

1. каждый режим получает 20 неучитываемых разогревочных шагов;
2. выполняются шесть зарегистрированных блоков;
3. каждый режим в каждом блоке содержит 50 измеряемых шагов;
4. [время на устройстве](../../docs/glossary.md#term-device-time)
   измеряется с помощью `ROCm events`;
5. синхронизация выполняется в конце режима блока;
6. внутри отдельных областей синхронизация не выполняется;
7. загрузка данных, контрольной точки и сериализация находятся вне измеряемого
   участка.

Одна подтверждающая [попытка](../../docs/glossary.md#term-attempt)
соответствует одному `model_seed` и выполняется в изолированном процессе.
Всего ожидается десять попыток.

## Первичный показатель

Для одного сбалансированного блока суммируются 50 измеряемых шагов:

```text
A = total device time baseline_no_observer
B = total device time calibration_observer
C = total device time live_attribution
K = sum of seven exclusive live region times
```

Сначала все компоненты сравниваются в абсолютном времени устройства:

```text
H = B - A
R = C - K
E = R - H = (C - K) - (B - A)
```

где:

- `H` — подписанная стоимость наблюдателя;
- `R` — подписанный неатрибутированный разрыв боевого режима;
- `E` — остаток после вычитания стоимости наблюдателя.

Нормирование выполняется один раз на чистое базовое время `A`:

```text
D = E / max(A, epsilon)
```

Таким образом, величины с разными знаменателями не вычитаются. Значения
`H`, `R`, `E` и `D` не обрезаются до нуля. Абсолютные величины публикуются
только как вторичная диагностика.

Для одной модели:

```text
D_seed = median(D over 3 batches and 6 order blocks)
```

Между десятью моделями первичная статистика — медиана `D_seed`.

Односторонняя верхняя 95-процентная граница вычисляется процентильным `bootstrap`:

```text
resampling unit = model_seed
bootstrap repeats = 10000
bootstrap seed = 20260716
```

Зарегистрированный запас:

```text
delta_excess = 0.01
```

`CAL-COST-MA1` проходит тогда и только тогда, когда:

```text
upper_95_bootstrap_bound(median(D_seed)) <= 0.01
```

Широкая неопределённость увеличивает верхнюю границу и тем самым затрудняет
прохождение.

Запас `delta_excess = 0.01` означает один процент чистого базового времени
`A`. Он предназначен только для остаточной ошибки замыкания стоимости
`SI-MA1` после локальной калибровки наблюдателя.

## Граница механизма оценки `ECZ`

`SI-MA1` не выполняет механизм оценки `ECZ`. Его стоимость не входит ни в
семь областей `K`, ни в `H`, ни в запас `delta_excess`.

Будущие `B1/B2` должны учитывать отдельные области управляющего контура:

```text
diagnostic_feature_acquisition
ecz_evaluation
action_selection
fallback_validation
```

В этих этапах полная дополнительная стоимость получения признаков, оценки
`ECZ`, выбора действия и проверки `fallback` должна сравниваться с реально
избежанным вычислением. Положительный `SI-MA1` не доказывает окупаемость
механизма `ECZ`.

## Барьеры

### `NUM-MA1`

Проходит, если `A` против `B` и `A` против `C` сохраняют `loss`, `final beliefs`, `parameter gradients`, `model state`, `inputs`, `targets` и `RNG` в пределах
зарегистрированных допусков ROCm/float32.

Этот барьер проверяет [отсутствие возмущения измерением](../../docs/glossary.md#term-non-perturbation).

### `TOPO-MA1`

Проходит, если `B` и `C` имеют одинаковую топологию наблюдателя:

- число `hooks`;
- число `events`;
- число записей областей;
- политику `saved tensors`;
- число `buffer writes`;
- число синхронизаций;
- `hash` конфигурации `instrumentation`.

### `BAL-MA1`

Проходит, если для каждой модели и пакета присутствуют все шесть
перестановок ровно по одному разу, а каждый режим дважды занимает каждую
позицию.

### `CAL-COST-MA1`

Проходит по первичному правилу верхней доверительной границы. Все времена и
блочные показатели должны быть конечными, неотрицательными там, где это
требуется формулой, и полными.

### `CMP-MA1`

Проходит при полной матрице, сохранении всех запусков, контрольных сумм и [происхождения артефактов](../../docs/glossary.md#term-provenance). Доступ к тестовой
выборке запрещён.

## Итоговое решение

```text
si_ma1_passed =
    prerequisites_verified
    and NUM-MA1
    and TOPO-MA1
    and BAL-MA1
    and CAL-COST-MA1
    and CMP-MA1
```

Состояния:

- `pass` — все зарегистрированные барьеры истинны;
- `fail` — доказательные материалы полны и хотя бы один барьер ложен;
- `inconclusive` — полнота нарушена по документированной инфраструктурной
  причине, а научный провал не установлен.

Это отдельное [решение о допуске](../../docs/glossary.md#term-decision-gate). Положительный результат
разрешает только отдельную предварительную регистрацию и реализацию `B1/B2`.
Он не разрешает активное управление вычислением.

## Вторичный анализ

Публикуются без права переопределить первичное решение:

- все `D_seed`;
- среднее и медиана;
- двусторонние `bootstrap`-интервалы;

## Политика повторов

Каждая [попытка](../../docs/glossary.md#term-attempt) имеет неизменяемый
идентификатор и сохраняется. Замена допускается только при
инфраструктурном отказе и при полном совпадении `source`, `image`, `contract`, `seed`, `batches` и `checkpoints`. Научный `fail` не является причиной замены.

## Граница интерпретации

При `pass` разрешён вывод только в зарегистрированной области:

> Неатрибутированный боевой временной разрыв не превышает независимо
> сопоставленную стоимость наблюдателя более чем на один процентный пункт
> по односторонней верхней 95-процентной границе на уровне моделей.

Не разрешаются:

- изменение исторического `COST-MA0=false`;
- универсальная стоимость наблюдателя;
- универсальные доли времени;
- использование запаса `delta_excess` как бюджета механизма оценки `ECZ`;
- утверждение, что `SI-MA1` измеряет время механизма оценки `ECZ`;
- разрешение пропуска вычислений;
- безопасность активного управления;
- минимальное достаточное фактор-пространство;
- пропорциональное ускорение;
- перенос на другой GPU, ROCm, архитектуру, набор данных или топологию
  наблюдателя.
